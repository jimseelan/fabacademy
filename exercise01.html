Ubuntu and the window focus

Setting up Ubuntu was not very difficult, but trying to solve the anooying window not focusing issue has taken the lion share of my initial week. BTW, I am a newbie to Ubuntu, and installed it on a 5 year old Lenovo T420 which my friend loaned me for the 6 months of Fabacademy. Windows is working okay on this T420, but for some strange reasons on Ubuntu, the mouse will not click on the front window. This issue will pop in after 2mins of logon and was a real annoyance. I tried searching the various forums and tried the various solutions. Few of the things I tried out are

  • Initially I thought its a mouse issue and done some search on that regard, but after few hours I could figure out that the issue was with the 'window focus'.
  • As mentioned here. I Installed the compizconfig-settings-manager, and changed the "Focus Prevention Level" from 'Low' to "Off". This seemed to solve the problem on Day-1, but unfortunatly the issue came back.
  • Then as mentioned here. I made the changes in Unity Tweak tool, but of no much use.
  • By this time I have figured that this issue sometime goes off on ots own; but returns after a few minutes.
  • Then via Terminal I tried Unity --replace which temporarily did a reset and a temporary relief.
  • Later I found that only Unity will also solve, but again momentarily only.
  • Later upon discussion with my peers here, I installed Cinnamon. I really liked the UI, but after 2 minutes of usage the window focus issue was back there too.
Unfortunately, none of the above solutions gave a permamnent fix to the issue. So I need to be get annoyed with Ubuntu till I get a new laptop. In the mean time I have managed to get the website made on windows using brackets. I then used Github to take it to Ubuntu, and then used git to push it to the fablab repository.

Git - Setting it up and using

Git is used as the version control system at the Fabacademy. The first fabacademy specific activity that I have done at the fablab is setting up GIT and using it. After Neils session, our local Instructor Yadu gave us a brief on Git and about the process of using and setting it up. GIT was entirely new to me and initially I was skeptical on my capabality to understand GIT, but after learning to use it I think I am having a fair idea of it.
I have also found an interesting simulator to try git without worrying about any wrong doings. The link of this git simulator is here

Steps followed to setup git

  • Create a fab lab account, and then create the gitlab account on git.fabacademy.org
  • Then install git by entering the following code in git. sudo apt-get install git
  • Then we need to generate the ssh keys by entering the following command in terminal ssh-keygen -t rsa -b 4096 -C "enter mail id"
  • The file name will be asked and enter it, or preferrably use the default location itself.
  • A passphrase will be asked them, and need to enter an easy to remember one.
  • Id_rsa is your private key and id_rsa is your public key
  • Then need to ensure that the ssh agent is enabled by entering eval "$ (ssh-agent -s)" #ensure ssh agent is enabled.
  • Then add the ssh key to the agent by ntering ssh-add ~/.ssh/id_rsa
  • Then global user id and global email ide needs to be added. The following commands are used for adding username git config --global user.name "Enter your username" and the following for the email git config --global user.email mail id
  • This user name needs to be given to instructor so that he could add us as users to the class repository
  • Then we need to enter the public key to the gitlab account by sudo gedit ~/.ssh/id_rsa.pub copy the entire contents fron the gedit window and close the window.Then paste this key to git.fabacademy.org under Profile Settings >> SSH keys >> Keys. Also need to ensure that a title is given to identify the PC used. And then click add key.
  • Then using terminal navigate to a directory, in my case I used the documennts folder. Then type the collowing to clone the repository git@git.fabacademy.org:fabacademy2017/fablabtrivandrum.git
